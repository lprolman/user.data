Backing up/Restoring Useful Evergreen Workstation Files

June 2, 2014 - 9:58am — ckolasinski

A few Evergreen workstation settings can be backed up in case of upgrade issues, workstation failure, or to be able to have settings be the same across workstations.     For example, the column configurations on the checkout, bills, item status screen, et al. can all be backed up and even transferred between workstations so that column configurations are the same across workstations.  Here are the settings that can be backed up:

  • Receipt templates †
  • Printer settings
  • Column configurations †
  • Offline circulation files

† These are files that can be safely transferred between workstations.

 

Backing up the files

First, access the computer’s hard drive and go to the following file location depending on operating system.  It is in these file locations that Evergreen stores workstation settings information.

Windows 7 and 8:

Local Disk (C:) => Users => USERNAME => AppData => Roaming => OpenILS => open_ils_staff_client => Profiles

Windows XP:

Local Disk (C:) => Documents and Settings => USERNAME => Application Data => OpenILS => open_ils_staff_client => Profiles

NOTE:  The folder name ‘AppData’ or ‘Application data’ may need to be typed into the file browser’s navigation bar if the ‘AppData’ folder is not present.

USERNAME in this case is the Windows username of the staff member who logs in and uses Evergreen.  Also, there should only be one folder under Profiles.  Enter this folder and this is where the settings files reside.  If more than one profile exists in the ‘profiles’ folder, feel free to back up each profile’s settings.

Next, back up the following files.   Most will be under the folder called ‘Chrome’ and these files are marked with the ‘chrome/’ prefix.  Each file has an explanation of what it actually does in the Evergreen staff client.

  • prefs.js                                                              //Mostly printer settings but many others, too
  • chrome/print_list_templates                          //Receipt Printer Templates
  • chrome/tree_column_for…                             //Column configurations
  • chrome/pending_xacts…                                //Offline circulation files

 

There may be zero or more of the ‘tree_column…’ files but it is important to back up all of the files that start with this phrasing in order to back up all column configurations. 

Be sure to only copy and paste these files to the backup location. Otherwise, the files may be removed from the Evergreen profile which may cause errors.

Once transferred to preferred backup media, the files can be kept for later use or transferred to another workstation.

 

 

Restoring/transferring of files

Restoring these files on a new/repaired workstation is fairly simple:

  1. Install Evergreen on new or repaired workstation (if not already installed)
  2. Register the workstation and log in once (unless previously registered and logged into)
  3. Exit Evergreen completely
  4. Copy and paste the above files from the backup media into the same directories as they were in before

The files that can be transferred between workstations should be done when Evergreen is closed.  On the next login, the new settings will be in place.

WARNING: It is not recommended that the prefs.js file be taken from one version of Evergreen and used in another.  For example, it is not recommended that the pref.js file from Evergreen 2.4 be transferred to a workstation that is set up for 2.5.  It is not recommended because these settings may change between versions and could cause errors if transferred.  

A Note about Offline Transaction File Naming Conventions

The offline transaction files can have three different states including ‘pending’, ‘complete’, and ‘exported’.  Each file state can be identified:

Pending Transaction files do not have an extension or numbers but simply say ‘pending_xacts’.  This file state will cause Evergreen to display ‘Offline Transactions Pending’ in a red box on the login screen.  If the file is removed or given an extension, the red box will be removed.  Pending offline transactions can be processed in Offline Transaction Management in Evergreen.

Uploaded Transactions have numbers and a ‘.complete’ extension.

Exported Transactions have the word ‘exported’ appended to the name and numbers at the end

 

 

* The folder name ‘AppData’ or ‘Application data’ may need to be typed into the file browser’s navigation bar if the ‘AppData’ or ‘Application data’ folder is not present. Click on the file navigation bar and type ‘/AppData’ or ‘/Application Data’ which should will allow access.
